Lesson 5: Tables

What are Tables?

Tables allow web developers to arrange data into a grid like structure. Aside from tables with grid border, you can also create tables without border, and position content without it looking like a grid.

    The essential elements for tables are:
  • <Table> creates the table
  • <tr> creates a row
  • <th> creates a heading cell
  • <td> creates a data cell

Table Borders

In HTML 5 we use CSS to create borders. If you wants a table with borders, add the following code to the CSS section of your web page on CodePen: Table, th, td {border:1px solid black;}

Table and Cell Size

By default, a table will size itself to fit the contents of the cell within it. But you can force a table (or the cells within it) to have specific sizes. You can specify pixels (px) or percentages of the width or height of the page. If you want to control the size, write the following into the styling section of CodePen:
Table{width:100%;height:500px}
th,td{width:100px;height:50px;}

Colspan and Rowspan

Sometimes you want a cell to break access to more than one row or column (or both). In this case, you use the rowspan or colspan attribute. ( the number tells how many of either it expands to)
<th colspan=”2” >
<td rowspan=”2” >

Lesson 6: Iframes

What are Iframes?

Iframes allow an HTML document to display the contents of one or more other HTML documents. We use the <iframe> element to accomplish this

HTML iframe Syntaz:

<iframe src=”url” title=”description” width=”” or height=””></iframe>
The src attribute is used to determine the path to the html document or web page that will provide the contents of the iframeIframes are inline tags; if the width of an iframe is less than the width of the page, other content may appear to the right or left of itThe title attribute is used to provide screen readers (an in case the source page is inaccessible) a description of the page that should be displayed in the iframe

Height and Width

Lesson 7: Head Elements

What is the "Head?"

The head is where the page’s metadata is found, along with everything else that is meant not to appear on the visible part of the page.

What Should Go inside the Head?

  • Title: the title is what shows up at the top of the web page window or tab
  • Metadata: data that is invisible to the viewer but is visible to the web crawlers and this important when the page appears in search results
  • Styles: instructions to the web browser on how the web page should look
  • Scripts: JavaScript code that must run before the body of the web page loads.
  • Links: important files that are linked to the page, including external CSS and JavaScript files.
