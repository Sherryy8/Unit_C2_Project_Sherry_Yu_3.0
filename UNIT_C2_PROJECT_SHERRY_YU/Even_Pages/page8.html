Lesson 12:Semantic Elements

What are Semantic Elements?

HTML achieves document organization through semantic markup. Semantic elements defines a meaning rather than a generic division– also makes it accessible to screen readers and other assistive technologies. There should be a hierarchy of headers (h1 to h6 are semantic tags.) Only one hi tag should be used within in each page. You can have as many h2 tags as you need, there should be at least one h2 for every h3, it is all like an outline. You don't want more than three levels of headings within a page. There can be one header element for every page but it has to follow the heading hierarchy– it is located at the top of the web page.

  • There can be more than one<nav>elements within a page, if you have more then one make sure each<nav>

  • a <nav> be have a heading, in this case the aria-label attribute would be used

  • There is only one<main>element per page
  • There can be one footer element per page just like the header element
  • <li>are also semantic elements
  • <div>element is not semantic because it is also a section it does explain what the section is talking about
  • <section> and <article>are semantic tags
  • <article>have a topic and might have content separate from the rest of the page
  • <aside>is a semantic element
  • <details>and <summary>work together and are semantic elements
  • <summary>is a subsection to details
  • <details> let you hide content and open it with the press of an arrow
